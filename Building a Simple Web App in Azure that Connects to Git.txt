Building a Simple Web App in Azure that Connects to Git

Steps (Long Method)
1. Create Resource > Web App
	- Create Resource Group
	- Instance > Name > [App Name]
		- Publish > Code
		- Runtime Stack > ASP.NET V4.8
	- App Service Plan
		- Sku and Size > [Choose plan (cheapest/free tier)]
	- Click "Review + Create"
	- Click "Create"
		- Once resource is created, click "Go to resource"
			- URL: [webappname].azure.websites.net
2. Run Azure Cloud Shell button (top column ">_" in square)
	- ACS: Terminal in browser
	- Use Bash (not PowerShell)
3. CLI commands
	1. git clone [URL] (access github repository)
	2. git config --global user.email "[email]"
	3. git config --global user.name "[first and last name]"
	4. cd [respository file path] (end with "/")
	5. ls (to lsit files in repository)
	6. git init (initialize git repository)
	7. git add . (add files to repository)
	8. git commit -m "[commit message]"
	9. az webapp deployment source config-local-git --name [web app name] --resource-group [resource group name] (returns URL for git deployment with embedded user-level credentials)
	10. az webapp deployment list-publishing credentials --name [web app name] --resource-group [resource group name] (returns link with username and password (scmUri))
		- Copy scmUri link
	11. git remote add azure [url (embeded in single quotes)]
	12. git push azure master (deploys code)
4. Go to web app link (should show finished HTML page)

Steps (Short Method)
3. CLI commands
	1. az webapp up --name [web app name] --html (tells web app not to try to interpret files)
		- Web app doesn't exist, so resource group, app service plan, app service app, etc. are automatically created
		- Creates zip package
4. Go to web app link (should show finished HTML page)
